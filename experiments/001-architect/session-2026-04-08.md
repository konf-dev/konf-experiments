## Findings

### Finding 001: Docker VOLUME directive conflicts with cap_drop ALL
The VOLUME directive in Dockerfile creates directories as root. With `cap_drop: ALL`, even root inside the container can't chown them. **Fix:** Don't use VOLUME directive. Let docker-compose mount handle it, and create directories with correct ownership in the Dockerfile before switching to non-root user.

### Finding 002: UID matching is critical for shared volumes
Docker creates files as the container user's UID. If this doesn't match the host user, you get permission denied. Ubuntu 24.04 base image has a `ubuntu` user at UID 1000. **Fix:** Rename the ubuntu user to konf-agent instead of creating a new one.
